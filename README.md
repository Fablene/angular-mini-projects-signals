# angular-mini-projects-signals
mini-projets angular

Contexte:
Développer une application de gestion de tâches simple, moderne et fluide. L’utilisateur peut créer, modifier, supprimer et filtrer ses tâches. Le but est de manipuler l’état local avec les signals d’Angular.


Fonctionnalités attendues:
Ajouter une tâche
Marquer une tâche comme terminée / non terminée
Supprimer une tâche
Modifier le texte d’une tâche
Filtrer les tâches :
-- toutes
-- actives
-- terminées
Afficher le nombre de tâches restantes
Bouton pour supprimer toutes les tâches terminées
Sauvegarde automatique dans le localStorage


Contraintes techniques:
Utiliser des standalone components
L’état principal doit être géré avec des signals
Les filtres doivent être calculés avec computed()
La sauvegarde locale doit passer par effect()


Pas de backend


Bonus
Ajouter une date de création
Ajouter une priorité
Réorganiser les tâches
Ajouter un mode sombre
